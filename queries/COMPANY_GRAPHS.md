# Query Bank: Company Graphs & Structured Context

## Domain
Knowledge graph architecture for organizations — structured context repositories, tacit knowledge externalization, company-as-graph patterns, and the arscontexta methodology.

**Thesis being tracked:** Structured markdown knowledge graphs (Obsidian-style, wikilink-traversable) are the optimal context architecture for agentic systems operating on organizational knowledge. The same structural properties that made codebases agent-readable (text files + explicit relationships) can be deliberately engineered for all knowledge work.

```

├── org/

├── teams/
│   ├── engineering/        # standards, architecture, runbooks

├── projects/
│   ├── product-alpha/
│   │   ├── prd/            # product spec as a graph of claims
│   │   ├── features/       # backlog → shipped lifecycle

│   │   └── decisions/      # architectural choices
│   └── product-beta/
├── research/               # deep domain knowledge graphs
├── transcripts/            # raw team conversations
├── archive/                # processed history
└── CLAUDE.md               # teaches the agent how your company works
```

## Connection to Systems Theory Thread

The Company Graph pattern directly addresses **ST-002 (governance speed asymmetry)**:
- Agents with structured context graphs operate at radically higher fidelity than agents with unstructured dumps
- This accelerates the capability curve without (yet) addressing the governance gap
- Counter-thesis: well-structured company graphs may also be the governance primitive — decisions with reasoning attached create the audit trail NIST requires

Connection to **ST-001 (near-critical cascade)**: unstructured organizational knowledge graphs create hidden dependencies that compound cascade risk at scale. Explicit graph structure with wikilinks = auditable dependency map.
