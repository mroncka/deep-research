# Query Banks

Queries are the intelligence fuel. Each query bank covers one research domain. Queries are designed to:

1. **Surface net-new signal** — not just confirm what's already known
2. **Cross-domain** — span primary sources, academic, news, and community
3. **Temporally anchored** — include `[current month year]` placeholder for freshness
4. **Layered** — primary query + 2–3 follow-up probes for depth

## Query Design Principles

### Temporal Anchoring
Always append `[MONTH YEAR]` or `newer_than:30d` style qualifiers. Stale queries surface stale results.

### Specificity Gradient
Start broad (domain sweep), then narrow (specific technology/event), then deep (mechanism/implication).

### Contradiction Seeking
> "What evidence contradicts [current dominant thesis]?"

This is the most underused query pattern. It forces confrontation with disconfirming evidence.

### Source Diversity
For each topic, deliberately query across:
- GitHub (activity, issues, releases)
- arXiv / academic preprints
- Community (HN, Reddit, Discord)
- Primary documentation (official announcements)
- Secondary analysis (Substack, Medium, blogs)
